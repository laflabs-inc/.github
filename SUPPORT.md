# Support

도움을 요청하기 전에 현재 문제의 성격을 구분해주세요.

## Bug

재현 가능한 Software Bug는 해당 Repository의 **Bug Report** Issue Form을 사용해주세요.

## Feature request

새로운 기능 또는 개선 제안은 **Feature Request** Issue Form을 사용해주세요.

## Questions

사용 방법이나 구현 질문은 해당 Repository가 제공하는 Documentation, Discussion 또는 공식 Support Channel을 먼저 확인해주세요.

## Security

Security Vulnerability는 공개 Issue로 등록하지 마세요. `SECURITY.md`를 따라 비공개로 보고해주세요.

## Service incidents

운영 중인 LafLabs 서비스의 장애, 결제, 계정, 고객 데이터와 관련된 문제는 공개 GitHub Issue가 아닌 해당 서비스의 공식 Support Channel을 이용해야 합니다.

공개 Issue에는 Credential, Access Token, 개인정보, Production Log 원문을 포함하지 마세요.
